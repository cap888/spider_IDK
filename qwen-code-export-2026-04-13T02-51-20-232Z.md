## Режимы WiFi подключения

### 1. **STA (Station Mode) — Подключение к существующей сети**

**Как работает:**
- ESP32 робота и ESP32 джойстика подключаются к вашему WiFi роутеру
- Общение через локальную сеть

**Плюсы:**
- Большая дальность (зависит от роутера)
- Можно управлять с любого устройства в сети
- Стабильное соединение
- Доступ к интернету (обновления, NTP)

**Минусы:**
- Нужен WiFi роутер рядом
- Зависимость от внешней инфраструктуры

---

### 2. **AP (Access Point Mode) — Робот как точка доступа**

**Как работает:**
- ESP32 на роботе создаёт свою WiFi сеть
- Телефон/джойстик подключается напрямую к роботу

**Плюсы:**
- Автономность — не нужен роутер
- Работает везде (парк, улица, дача)
- Прямое соединение, меньше задержка
- Нет зависимости от внешней сети

**Минусы:**
- Меньшая дальность (~30-50m)
- К роботу нельзя подключить другие устройства одновременно
- Нужно переключать WiFi на телефоне

---

### 3. **AP + STA (Комбинированный режим) — Лучшее решение**

**Как работает:**
- ESP32 создаёт свою точку доступа И одновременно подключается к WiFi
- Можно управлять и с телефона (через AP), и с ПК (через STA)

**Плюсы:**
- Максимальная гибкость
- Работает автономно и в доме
- Можно перенастраивать удалённо

**Минусы:**
- Чуть сложнее код
- Выше энергопотребление

---

## Моя рекомендация

### **Для мобильного робота: AP Mode (точка доступа)**

**Причины:**
1. Робот будет работать везде, не только дома
2. Не нужно таскать роутер на природу
3. Меньше задержка (прямое соединение)
4. Проще настройка для пользователя

**Реализация:**
- ESP32 на роботе создаёт сеть "SpiderRobot" с паролем
- Открываете на телефоне браузер → `192.168.4.1`
- Видите виртуальный джойстик (web-интерфейс)
- Или подключаете физический джойстик на ESP32 к той же сети

   * @param target_x Целевая позиция X (мм)
   * @param target_y Целевая позиция Y (мм)
   * @return Angles {base_angle, joint_angle} в градусах
   */
  LegAngles inverseKinematics(float target_x, float target_y) {
    LegAngles angles;
    
    // Расстояние от базы до цели
    float r = sqrt(target_x * target_x + target_y * target_y);
    
    // Проверка достижимости
    float max_reach = l1 + l2;
    if (r > max_reach) {
      #ifdef DEBUG_KINEMATICS
      Serial.println("IK: Target out of reach!");
      #endif
      
      // Возврат к максимально возможной позиции
      r = max_reach * 0.95;
    }
    
    if (r < 1) {
      r = 1;  // Защита от деления на ноль
    }
    
    // Угол до цели (относительно базы)
    float gamma = atan2(target_y, target_x);
    
    // Теорема косинусов для угла сустава
    float cos_theta2 = (target_x * target_x + target_y * target_y - l1 * l1 - l2 * l2) / (2 * l1 * l2);
    cos_theta2 = constrain(cos_theta2, -1.0, 1.0);
    float theta2 = acos(cos_theta2);
    
    // Угол сустава (в градусах)
    angles.joint_angle = radToDeg(theta2);
    
    // Угол первого сегмента
    float cos_theta1 = (l1 * l1 + r * r - l2 * l2) / (2 * l1 * r);
    cos_theta1 = constrain(cos_theta1, -1.0, 1.0);
    float theta1 = acos(cos_theta1);
    
    // Базовый угол (в градусах)
    float base_rad = gamma + theta1;
    angles.base_angle = radToDeg(base_rad) + 90;  // Смещение для сервопривода
    
    // Ограничение углов
    angles.base_angle = constrain(angles.base_angle, LEG_BASE_ANGLE_MIN, LEG_BASE_ANGLE_MAX);
    angles.joint_angle = constrain(angles.joint_angle, LEG_JOINT_ANGLE_MIN, LEG_JOINT_ANGLE_MAX);
    
    #ifdef DEBUG_KINEMATICS
    Serial.print("IK: x=");
    Serial.print(target_x, 1);
    Serial.print(" y=");
    Serial.print(target_y, 1);
    Serial.print(" -> base=");
    Serial.print(angles.base_angle, 1);
    Serial.print(" joint=");
    Serial.println(angles.joint_angle, 1);
    #endif
    
    return angles;
  }
  
  /**
   * Проверка достижимости позиции
   */
  bool isReachable(float x, float y) {
    float r = sqrt(x * x + y * y);
    return r <= (l1 + l2) * 0.95;  // 95% от максимальной досягаемости
  }
  
  /**
   * Расчет рабочей зоны лапы
   * Возвращает максимальный радиус
   */
  float getMaxReach() {
    return l1 + l2;
  }
  
  /**
   * Получение длин сегментов
   */
  float getL1() { return l1; }
  float getL2() { return l2; }
  
  /**
   * Установка длин сегментов (если конструкция изменилась)
   */
  void setSegmentLengths(float new_l1, float new_l2) {
    l1 = new_l1;
    l2 = new_l2;
  }

## 1. ПУЛЬТ УПРАВЛЕНИЯ

### Джойстик KY-023 → Arduino Nano
```
KY-023          Arduino Nano
------          ------------
VRx      →      A0            (ось X)
VRy      →      A1            (ось Y)
SW       →      D2            (кнопка, INPUT_PULLUP)
+5V      →      5V
GND      →      GND
```

### Bluetooth HC-05 → Arduino Nano
```
HC-05           Arduino Nano
-----           ------------
TX       →      D3            (SoftwareSerial RX)
RX       →      D4            (SoftwareSerial TX)
VCC      →      5V
GND      →      GND
```

**Важно:** HC-05 должен быть в режиме **Master** для пульта.

### Настройка HC-05 в режим Master
1. Подключите HC-05 к USB-TTL адаптеру
2. Войдите в AT-режим (удерживая кнопку при подаче питания)
3. Отправьте команды (скорость 38400):
   ```
   AT                  → OK
   AT+ROLE=1           → OK (Master mode)
   AT+PSWD="1234"      → OK (пароль)
   AT+BIND=<address>   → OK (адрес HC-05 на роботе)
   ```

### Протокол Bluetooth

Формат пакета (6 байт):
```
[0xFF][Joy X 0-255][Joy Y 0-255][Buttons][Checksum][0xFE]
```

- **0xFF**: Стартовый маркер
- **Joy X**: Положение джойстика по X (128 = центр)
- **Joy Y**: Положение джойстика по Y (128 = центр)
- **Buttons**: Битовая маска кнопок
- **Checksum**: Контрольная сумма (сумма байтов & 0xFF)
- **0xFE**: Конечный маркер

### Походка Tripod

Лапы делятся на две группы:
- **Группа A**: Лапы 0, 2, 4
- **Группа B**: Лапы 1, 3, 5

**Фаза 1:**
- Группа A: в воздухе (swing) → перемещается вперёд
- Группа B: на земле (stance) → толкает тело назад

**Фаза 2:**
- Группа B: в воздухе (swing) → перемещается вперёд
- Группа A: на земле (stance) → толкает тело назад

Это обеспечивает стабильность и плавность движения.
