# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## О проекте

Прошивка для шестилапого автономного робота на Arduino UNO с 12 сервоприводами Feetech FS90 (6 лап × 2 сустава). При включении робот выполняет фиксированную последовательность движений без пульта управления.

## Загрузка прошивки

```
# Открыть receiver/receiver.ino в Arduino IDE
# Выбрать плату: Arduino Uno
# Выбрать порт: соответствующий COM-порт
# Нажать «Загрузить»
```

Для отладки открыть Serial Monitor на скорости **115200 baud**.

## Включение отладки

В `receiver/Config.h` раскомментировать нужные флаги:
```cpp
#define DEBUG_SERVOS       // Позиции сервоприводов
#define DEBUG_KINEMATICS   // Расчёты кинематики
#define DEBUG_GAIT         // Состояние походки
```

## Архитектура

Поток управления при работе:
```
receiver.ino → RobotController::update()
                    ↓
             processSequence()   — конечный автомат состояний
                    ↓
             GaitEngine::update(dx, dy)  — расчёт целевых позиций лап
                    ↓
             LegKinematics::inverseKinematics(x, y)  — обратная кинематика
                    ↓
             ServoController::setLegAngles()  — задать целевой угол
                    ↓
             ServoController::update()  — плавное движение к цели
```

### Слои

| Файл | Ответственность |
|------|----------------|
| `receiver.ino` | Точка входа: `setup()` / `loop()` |
| `RobotController.h` | Конечный автомат автономной последовательности |
| `GaitEngine.h` | Походка Tripod — расчёт целевых позиций всех лап |
| `LegKinematics.h` | Прямая/обратная кинематика 2-суставной лапы |
| `ServoController.h` | Плавное движение сервоприводов через `Multiservo.h` (I2C) |
| `Config.h` | Все параметры проекта |

### Конечный автомат (RobotController)

Состояния выполняются строго по порядку:
`STATE_BLINK` → `STATE_FORWARD` → `STATE_PAUSE` → `STATE_BACKWARD` → `STATE_PAUSE` → `STATE_LEFT_STRAFE` → `STATE_PAUSE` → `STATE_RIGHT_STRAFE` → `STATE_ROTATE` → `STATE_STOP`

Переходы обрабатываются в `processSequence()`. Чтобы добавить новое движение — добавить состояние в enum `RobotState` и обработать его там же.

### Походка Tripod

Лапы делятся на две группы, которые чередуют фазы:
- **Группа A**: лапы 0, 2, 4
- **Группа B**: лапы 1, 3, 5

`GaitEngine::update(dx, dy)` принимает нормализованное направление движения (−1.0 .. 1.0). Поворот задаётся отдельно через `setRotation(turn)` и суммируется с трансляцией.

### Кинематика лапы

Каждая лапа — 2-суставной манипулятор в плоскости XY:
- `l1` = `LEG_BASE_LENGTH` = 30 мм
- `l2` = `LEG_JOINT_LENGTH` = 50 мм

Обратная кинематика (`inverseKinematics`) реализована через теорему косинусов; при выходе за рабочую зону позиция ограничивается до 95% максимального радиуса.

### Индексация сервоприводов

В `ServoController` каждая лапа занимает 2 канала Multiservo Shield: `leg * 2` — базовый сервопривод, `leg * 2 + 1` — суставной. Итого используются каналы 0–11 (из 18 доступных). Все команды передаются по I2C, Arduino не задействует PWM-пины для сервоприводов.

## Ключевые параметры (Config.h)

- `INITIAL_POSITIONS[6]` — калибровочные углы для каждой лапы; при неверной сборке корректировать здесь.
- `LEG_HOME_X / LEG_HOME_Y` — домашняя позиция конца лапы в мм; влияет на устойчивость стойки.
- `CONTROL_LOOP_INTERVAL` = 20 мс — частота главного цикла управления (50 Гц).
- `SERVO_MOVE_SPEED` — максимальная скорость сервоприводов (°/с); уменьшить при рывках.

## Аппаратные зависимости

- Библиотека **Multiservo** от Амперки (установить через Library Manager или `amperka/Multiservo` на GitHub).
- Встроенные: `Wire.h` (I2C), `math.h`.
- Шилд Multiservo Shield v2 управляется по I2C (адрес `0x47`), устанавливается непосредственно на Arduino UNO; физического подключения пинов PWM не требуется.
- Питание сервоприводов — **отдельный** источник 5V 3A через разъём шилда; нельзя брать от пина 5V Arduino.
