# 🕷️ Spider Robot - Шестилапый робот на Arduino

Автономный шестилапый робот с 12 сервоприводами Feetech FS90, выполняющий запрограммированную последовательность движений при включении.

## Особенности

- ✅ **12 сервоприводов Feetech FS90** (6 лап × 2 сустава)
- ✅ **Автономный режим** — выполняет последовательность движений при включении
- ✅ **Походка Tripod** (чередующиеся тройки лап)
- ✅ **Плавные движения** без рывков
- ✅ **Обратная кинематика** для точного позиционирования
- ✅ **Модульная архитектура** для лёгкой модификации

## Автономный режим

При включении робот выполняет следующую последовательность:

1. **Задержка** — 2 секунды (LED мигает с частотой 3 Гц)
2. **3 шага вперёд**
3. **Пауза** — 0.5 секунды (LED мигает с частотой 1 Гц)
4. **3 шага назад**
5. **Пауза** — 0.5 секунды
6. **3 стрейфа влево**
7. **Пауза** — 0.5 секунды
8. **3 стрейфа вправо**
9. **Поворот** — 360° против часовой стрелки (60°/сек, ~6 сек)
10. **Стоп** — робот останавливается

```
Временная диаграмма:
0 сек       — включение, LED мигает (3 Гц)
2 сек       — LED выключен, старт движения
2-5 сек     — 3 шага вперёд
5-5.5 сек   — пауза (1 Гц)
5.5-8.5 сек — 3 шага назад
8.5-9 сек   — пауза (1 Гц)
9-12 сек    — 3 стрейфа влево
12-12.5 сек — пауза (1 Гц)
12.5-15.5 сек — 3 стрейфа вправо
15.5-21.5 сек — поворот 360°
21.5 сек    — СТОП
```

## Настройка

Все параметры в `receiver/Config.h`:

```cpp
// Автономный режим
#define STARTUP_DELAY 2000       // Задержка при включении (мс)
#define STEP_COUNT 3             // Количество шагов
#define ROTATION_SPEED 60        // Скорость поворота (градусов/с)
#define LED_BLINK_FREQ_START 3   // Частота мигания LED при включении (Гц)
#define LED_BLINK_FREQ_RUN 1     // Частота мигания LED при паузе (Гц)
#define PAUSE_DURATION 500       // Пауза между шагами (мс)

// Геометрия лап
#define LEG_BASE_LENGTH 30    // Длина первого сегмента (мм)
#define LEG_JOINT_LENGTH 50   // Длина второго сегмента (мм)

// Параметры походки
#define STEP_HEIGHT 20        // Высота подъёма лапы (мм)
#define STEP_LENGTH 30        // Длина шага (мм)
#define STEP_FREQUENCY 2      // Частота шагов (Гц)

// Скорости
#define SERVO_MOVE_SPEED 60   // Скорость сервоприводов (°/с)
```

## Сборка и запуск

### 1. Сборка робота
1. Установите Multiservo Shield v2 на Arduino Uno (I2C соединяется автоматически через контакты шилда)
2. Подключите 12 сервоприводов к каналам 0–11 Multiservo Shield (лапа N: каналы N×2 и N×2+1)
3. Подключите BEC 5V 3A к разъёму питания шилда для сервоприводов

### 2. Загрузка прошивки
```bash
# Откройте receiver/receiver.ino в Arduino IDE
# Выберите плату: Arduino Uno
# Загрузите
```

### 3. Запуск
1. Подайте питание на робота
2. LED начнёт мигать (3 Гц) в течение 2 секунд
3. Робот выполнит последовательность движений
4. После поворота 360° робот остановится

## Архитектура

### Принцип работы

```
[Включение]
    ↓
[Robot Controller]
    ↓ (конечный автомат состояний)
[Gait Engine]
    ↓ (расчёт походки)
[Leg Kinematics] → обратная кинематика
    ↓ (углы сервоприводов)
[Servo Controller] → плавное движение
    ↓ (I2C команды)
[Multiservo Shield v2]
    ↓ (12 каналов PWM)
[12x Feetech FS90]
```

### Состояния автомата

- `STATE_BLINK` — мигание LED во время задержки
- `STATE_FORWARD` — STEP_COUNT шагов вперёд
- `STATE_PAUSE` — пауза между движениями
- `STATE_BACKWARD` — STEP_COUNT шагов назад
- `STATE_LEFT_STRAFE` — STEP_COUNT стрейфов влево
- `STATE_RIGHT_STRAFE` — STEP_COUNT стрейфов вправо
- `STATE_ROTATE` — поворот 360° против часовой стрелки
- `STATE_STOP` — конец

### Походка Tripod

Лапы делятся на две группы:
- **Группа A**: Лапы 0, 2, 4
- **Группа B**: Лапы 1, 3, 5

**Фаза 1:**
- Группа A: в воздухе (swing) → перемещается вперёд
- Группа B: на земле (stance) → толкает тело назад

**Фаза 2:**
- Группа B: в воздухе (swing) → перемещается вперёд
- Группа A: на земле (stance) → толкает тело назад

Это обеспечивает стабильность и плавность движения.

## Устранение проблем

### Сервоприводы не двигаются
- ✅ Проверьте питание сервоприводов на Multiservo Shield (5V от внешнего BEC)
- ✅ Убедитесь, что шилд плотно установлен на Arduino Uno
- ✅ Установите библиотеку Multiservo (`amperka/Multiservo`)

### Робот двигается неправильно
- ✅ Откалибруйте сервоприводы (установите в 90°)
- ✅ Проверьте `INITIAL_POSITIONS` в Config.h
- ✅ Убедитесь в правильности подключения к каналам Multiservo Shield

Подробности: `docs/wiring_diagram.md`

## Безопасность

⚠️ **ВАЖНО:**
- Используйте отдельный источник питания 5V 3A для сервоприводов (не от пина 5V Arduino!)
- Суммарный ток 12x FS90 может достигать 3A
- Всегда объединяйте GND всех источников питания
